SpringCloud系列-服务的注册和发现Eureka

​ “Eureka”来源于古希腊词汇，意为“发现了”。在软件领域，Eureka是Netflix在线影片公司开源的一个服务注册和发现组件，

和其他的Netflix公司的服务组件（例如负载均衡，熔断器，网关等）一起，被Spring Cloud社区整合为Spring Cloud Netflix模块。

1.Eureka简介

​ 和Zookeeper类似，Eureka是一个用于服务注册和发现的组件，最开始主要应用与亚马逊公司的云计算服务平台AWS，Eureka

分为Eureka Server和Eureka Client，Eureka Server为Eureka服务注册中心，Eureka Client为Eureka客户端。

​ Eureka主要涉及到三大角色：服务提供者、服务消费者、注册中心。

2.pom依赖

1.注册中心依赖

<dependency>
	<groupId>org.springframework.cloud</groupId>
	<artifactId>spring-cloud-starter-netflix-eureka-server</artifactId>
</dependency>

2.服务提供者(上游微服务) 和 消费者(下游微服务) 依赖

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>

3.注解

//声明在Eureka服务端，开启Eureka注册服务器
@EnableEurekaServer //声明当前应用是Eureka server
//声明在Eureka客户端，用于开启Client服务
@EnableEurekaClient //只针对Eureka
@EnableDiscoveryClient //兼容性强，适合其他注册发现中心的客户端

4.applicaton.yml

大致意思如注释:

server:
  port: 9004
spring:
  application:
    name: eureka-server
eureka:
  client:
    service-url:
      # eureka 服务地址，如果是集群的话；需要指定其它集群eureka地址
      defaultZone: http://127.0.0.1:9004/eureka
    # 表示不注册自己
    register-with-eureka: false
    # 不拉取服务
    fetch-registry: false
  instance:
    # 每个服务心跳的间隔时间
    lease-renewal-interval-in-seconds: 30
    # eureka多久没有接收到服务心跳销毁
    lease-expiration-duration-in-seconds: 90
  server:
    enable-self-preservation: false # 关闭⾃我保护模式（缺省为打开）生产环境可以使用，开发需要关闭
    eviction-interval-timer-in-ms: 1000 # 扫描失效服务的间隔时间（缺省为60*1000ms）

5.Eureka的自我保护

​ 当有一个新的Eureka Server出现时，他尝试从相邻的Peer节点获取所有服务实例注册信息。如果从相邻的Peer节点获取信息时出现了故障，

Eureka Server会尝试其他的Peer节点。如果Eureka Server能够成功获取所有的服务实例信息。则根据配置信息设置服务续约的阈值。在任何时间，

如果Eureka Server接收到的服务续约低于为该值配置的百分比（默认为15分钟内低于85%），则服务器开启自我保护模式，即不再剔除注册列表的信息。

这样做的好处在于，如果Eureka Server自身的网络问题而导致Eureka Client无法续约，Eureka Client的注册列表信息不再被删除，也就是Eureka Client还可以

被其他服务消费。

6.Eureka Server集群

​ Eureka Server不但需要接收服务的心跳，用来检测服务是否可用，而且每个服务会定期会去Eureka申请服务列表的信息，

当服务实例很多时，Eureka中的负载就会很大，所以必须实现Eureka服务注册中心的高可用，一般的做法是将Eureka Server集群化。

6.1 配置文件

---
#activate on profile :激活在概要文件
spring:
  config:
    activate:
      on-profile: peer1
server:
  port:
    9003
eureka:
  instance:
    hostname: peer1
  client:
    service-url:
      defaultZone: http://peer2:9004/eureka,http://peer3:9005/eureka
---

---
#activate on profile :激活在概要文件
spring:
  config:
    activate:
      on-profile: peer2
server:
  port:
    9004
eureka:
  instance:
    hostname: peer2
  client:
    service-url:
      defaultZone: http://peer1:9003/eureka,http://peer3:9005/eureka
---

---
#activate on profile :激活在概要文件
spring:
  config:
    activate:
      on-profile: peer3
server:
  port:
    9005
eureka:
  instance:
    hostname: peer3
  client:
    service-url:
      defaultZone: http://peer1:9003/eureka,http://peer2:9004/eureka
---

注：
●在yaml单一配置文件中，可用连续三个连字号（—）区分多个文件。
●Spring Boot2.4.x使用spring.config.activate.on-profile代替原来的spring.profiles

6.2 域名解析

使用peer1，peer2，peer3 代替 127.0.0.1

所以需要修改本地的host文件，c:\Windows\System32\drivers\etc\hosts，代码如下。

127.0.0.1 peer1
127.0.0.1 peer2
127.0.0.1 peer3

或者使用工具switchhost https://blog.csdn.net/libusi001/article/details/108516673

6.3 启动并测试

java -jar 04_cloud_eureka-0.0.1-SNAPSHOT.jar --spring.profiles.active=peer1
java -jar 04_cloud_eureka-0.0.1-SNAPSHOT.jar --spring.profiles.active=peer2
java -jar 04_cloud_eureka-0.0.1-SNAPSHOT.jar --spring.profiles.active=peer3

6.4 启动支付服务，支付微服务仅向9004的Eureka Server注册，代码如下。

eureka:
  client:
    service-url:
      defaultZone: http://peer1:9003/eureka,http://peer2:9004/eureka,http://peer3:9005/eureka
